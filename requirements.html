User Interviews

We conducted interviews in order to gather requirements for the two main different user types - admins and users.

Why will this ClinicalKitQR be useful?

We would like staff to be able to learn to use equipment on the job. This may aid gaps in knowledge, such as staff returning from leave, skills not being used for some time, etc. Essentially implementing Just In Time (JIT) learning.

What features do you look for on the website?

It would be great for users to log into their organization with a password. Admin privileges are also a must – we need read/write access for editing resources. Each type of equipment should have its own QR code generated for easy linking to resources.

How will QR codes be useful?

It is a consistent and useful way of tagging equipment. It would be able to link them to resources in Video, PDF, and other various formats. These can be easily accessed from mobile devices for hands-on use.

What features are you looking for website?

It would be great for it to be simple, but attractive. Also, for it to have easy navigation for quick access of resources.

What value would the ClinicalKitQR add to your role?

The quick access with QR codes to the right resources will help build and refresh my knowledge from face-to-face learning. Some resources may include setup of equipment, how to use it and its function – this will be vital in a neonatal unit

What value would the ClinicalKitQR add to your role?

Each type of equipment will link to a page with resources of different topic. So, if I had 10 of 1 type of ventilator, there would be one QR code for that ventilator type on each of them. This will reduce confusion and will make site navigation easy.
