Requirements

Project Background and Client Introduction

The Clinical QR Kit is a widely applicable solution to many departments within the NHS. It is a web application which utilises QR codes for ease of access of resources regarding equipment found in clinical departments, hospitals and other medical spaces. Scanning the QR codes tagged on equipment will take users to the web application. They will be prompted to login and can simply view the resources of different formats on the comfort of their mobile devices. Admin staff, being hospital admins and trust admins, can manipulate these resources, users, create more user groups, and share resources between hospitals.

Staff within the NHS struggle with retaining information on equipment they use, due to the vast range the field offers. The Clinical QR Kit offers a solution in which information can be accessed quickly without needing to revisit training, asking other colleagues who may not be readily available, and searching through study books; using Just In Time learning, staff learn on the job, working efficiently and reducing the likelihood of adverse consequences due to inaccessibility of information.

The recent COVID-19 pandemic has shown an increased use of QR codes, namely for the Test and Trace methods of the NHS. This contactless use of information input has shown great use in the pandemic, and the clients of the project wish to see this come to use within the medical space.

Project Goal

After meeting with our client and project partners, we understood that this project needed to be accessible from a range of devices. Implementing a website was the best choice. We also needed to create a database to store the resources the website would include. Lastly, each equipment type needed a QR code - this meant that QR code generation was key for the solution.

User Interviews

We conducted interviews in order to gather requirements for the two main user types - admins and users. This was necessary to understand more of the user's needs and ensure our solution was widely usable.

Why will this ClinicalKitQR be useful?

We would like staff to be able to learn to use equipment on the job. This may aid gaps in knowledge, such as staff returning from leave, skills not being used for some time, etc. Essentially implementing Just In Time (JIT) learning.

What features do you look for on the website?

It would be great for users to log into their organization with a password. Admin privileges are also a must – we need read/write access for editing resources. Each type of equipment should have its own QR code generated for easy linking to resources.

How will QR codes be useful?

It is a consistent and useful way of tagging equipment. It would be able to link them to resources in Video, PDF, and other various formats. These can be easily accessed from mobile devices for hands-on use.

What features are you looking for website?

It would be great for it to be simple, but attractive. Also, for it to have easy navigation for quick access of resources.

What value would the ClinicalKitQR add to your role?

The quick access with QR codes to the right resources will help build and refresh my knowledge from face-to-face learning. Some resources may include setup of equipment, how to use it and its function – this will be vital in a neonatal unit

What value would the ClinicalKitQR add to your role?

Each type of equipment will link to a page with resources of different topic. So, if I had 10 of 1 type of ventilator, there would be one QR code for that ventilator type on each of them. This will reduce confusion and will make site navigation easy.

MoSCoW List

The following list illustrates the MoSCoW requirements of our project. Having finalised our requirement gathering, we were able to create requirements with the MosCoW method. These requirements have also been updated and added to throughout development.

  • Users must be able to scan and be taken to login first before accessing resources

  • Users must be able to see what the admin has authrorised as being relevant to work in their trust

  • Two types of users: Clinical Staff and Admins

  • There are two types of admins: hospital admins and trust admins

  • A database must be created to manage the solution

  • QR codes must be generated on the creation of a new equpment

  • Resources must be able to be uploaded to the site

  • Any admin can share resources across the entire platform

  • New resources can be pushed to other administrators as a notification

  • There is a system to report problems with equipment resources

  • A way to report what equipment has been viewed by user groups

  • An advanced search to search by different criteria that is not just the equipment name and generated code

  • An FAQ section for all users

  • A contact book of all admins in the trust

  • Access to data of patient currently in hospital

  • Hold any sensitive data pertaining patients, employees and workers
